Phase 0 — Intake & Triage

Accept, classify, deduplicate, and prioritize topic submissions

✉
Intake Agent
Agent
Receives topic submissions from multiple entry points. Checks against existing topic registry to prevent duplicates. Classifies topic type. Assigns priority based on Slack discussion volume, deal attachment, and leadership asks.
Inputs
Slack commandCLI /intakeCLI /articleMaintenance triggers
Outputs
Topic Queue entryPriority scoreDuplicate flag
Checks EnablementHub/index.html for existing topics (live + planned)
Searches Slack for volume of discussion to gauge demand
Classifies: New Topic, Update Existing, or Out of Scope
Writes topic entry to topics-queue.json with metadata

Phase 1 — Research

Deep parallel research across all internal and external sources

🔍
Research Agent
Agent
Highest-leverage agent. Runs parallel searches across Slack, Google Drive, Salesforce, and public web. Ranks sources by authority (BRD > Canvas > Slides > Thread > Blog > Web). Identifies SMEs. Produces structured research report with gap analysis.
Data Sources
Slack canvasesSlack threadsGoogle DocsGoogle SlidesGoogle SheetsSalesforce SOQLPublic web
Outputs
Research ReportSource RankingSME ContactsGap AnalysisCompetitive Intel
Slack: canvases (type:canvases), threads, files — reads top canvases in full
Google Drive: BRDs, architecture docs, diagram inventory from slides
Salesforce: opportunities, product records, customer references
Web: official docs, Trailhead, release notes, competitor positioning
SME identification: who wrote the most content? That's the validation contact
Gap report: topics discussed in Slack with no official docs
Human checkpoint: After research, the agent presents findings. User reviews and approves before assembly begins.
2

Phase 2 — Content Assembly

Transform research into a production-quality field guide

✎
Assembly Agent
Agent
Writes the field guide. Uses research report as input. Selects structure based on topic type. Each topic gets unique layout.
Reads AgentFabric/index.html as style reference
Selects tab structure (4 tabs standard)
Writes copy: one-liner, pitch cards, compare splits, proof points
No two guides share the same section arrangement
Embeds diagrams from Diagram Harvester skill (shared with PDF Agent)
🖼
Diagram Harvester
Shared Skill
Extracts visual assets from Google Slides, Canvases, and Docs. Used by Assembly Agent and PDF Agent.
Gets slide thumbnails via get_page_thumbnail
Converts canvas diagrams to inline SVG/HTML
Stores source reference for freshness checks
3

Phase 3 — Testing & Validation

Automated quality gate — post-build and 2-3x/week schedule

✓
Testing Agent
Scheduled 2-3x/week
Quality guardian. Runs post-build and on schedule across all deployed topics. Failures block deployment; warnings are logged.
Link validation: HTTP check every href — 404s, 5xx, redirects
Internal URL scan: regex for slack.com, org62, *.sfdc.net (shared with PDF Agent)
Fact checking: cross-reference with help.salesforce.com
Diagram freshness: source slides modified since embed?
Content completeness: one-liner, tabs, FAQ, proof points required
Hub consistency: sidebar links match deployed URLs
Diff reporting: what changed vs. last run on scheduled checks
4

Phase 4 — Deployment

Push to Heroku, wire into Hub, verify end-to-end

🚀
Deploy Skill
Skill
Discrete deployment operation. Git init, Heroku create, push, verify HTTP 200, wire into Hub. Supports rollback.
git init + commit all files
heroku create sf-{topic-slug}
Push and wait for build
Verify HTTP 200 on deployed URL
Update EnablementHub sidebar, card, count, changelog (Hub Wiring — shared with Maintenance Agent)
Rollback on failure or post-deploy test issues
5

Phase 5 — FAQ Maintenance

Continuously mine Slack for new questions and keep FAQs fresh

?
FAQ Agent
Daily
Searches Slack for questions about each deployed topic (uses shared Slack Miner skill). Deduplicates, ranks, and presents FAQ candidates for user approval. Does NOT auto-modify live guides.
Question mining: Slack search for "?", "how do I", "what is" + topic keywords
Canvas extraction: parse informal Q&A sections
Deduplication: cluster similar questions, best phrasing
Answer sourcing: draft answers from threads and guide content
Human review: presents ranked candidates — user approves before injection
6

Phase 6 — Customer-Facing PDF

Dynamic, unique PDFs with AE customization

📄
PDF Agent
Agent
Generates unique branded customer PDFs per topic. Every PDF uses a different layout. AE can customize. Internal links scrubbed. Uses shared Diagram Harvester and Internal URL Scanner skills.
Dynamic layout: architecture → diagrams first; competitive → tables first
AE customization: sections, follow-ups, account context
Internal link scrubber: shared with Testing Agent
Diagram selection: shared Diagram Harvester picks most relevant
Version tracking: content hash flags stale PDFs
7

Phase 7 — Monitoring & Maintenance

Keep the entire Hub healthy, fresh, and evolving

📈
Maintenance Agent
Weekly
System immune system. Checks health of every deployed topic weekly. Detects content decay, usage drop-off, integrity issues. Triggers re-research for stale topics.
Usage analytics: Heroku logs — traffic vs. dead topics
Content decay: 60 days no update + new Slack discussions → flag
Hub integrity: shared skill with Testing Agent — sidebar links, counts
Dependency monitoring: SF release calendar → flag affected topics
Refresh trigger: push stale topics back into Intake Queue
Weekly digest: healthy / needs refresh / new FAQs added
